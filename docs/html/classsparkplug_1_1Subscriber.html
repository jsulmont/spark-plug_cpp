Detailed Description

Sparkplug B subscriber for consuming edge node messages with validation.

The Subscriber class receives and validates Sparkplug B messages:

  • NBIRTH/DBIRTH: Birth certificates
  • NDATA/DDATA: Data updates
  • NDEATH/DDEATH: Death certificates
  • STATE: Primary application state (optional)
  • Automatic sequence number validation (detects packet loss)
  • Node state tracking per edge node
Thread Safety
This class is NOT thread-safe. The callback is invoked on the MQTT thread.
Example Usage
auto callback = [](const sparkplug::Topic& topic,
const auto& payload) {
std::cout << "Received: " << topic.to_string() << "\n";
for (const auto& metric : payload.metrics()) {
std::cout << metric.name() << " = " << metric.double_value() << "\n";
}
};
sparkplug::Subscriber::Config config{
.broker_url = "tcp://localhost:1883",
.client_id = "my_subscriber",
.group_id = "Energy"
};
sparkplug::Subscriber subscriber(std::move(config), callback);
subscriber.connect();
subscriber.subscribe_all(); // Subscribe to all messages in group
// Keep running to receive messages...

void sparkplug::Subscriber::set_command_callback ( CommandCallback callback)

Sets a callback for receiving command messages (NCMD/DCMD).

Commands are messages sent from SCADA/Primary Applications to edge nodes to trigger actions like rebirth, reboot, or custom operations.

Parameters
callbackFunction to call when a command is received
Note
The callback is invoked on the MQTT client thread.
Common commands: "Node Control/Rebirth", "Node Control/Reboot", etc.
Example Usage
subscriber.set_command_callback([&publisher](const sparkplug::Topic& topic,
const auto& payload) {
for (const auto& metric : payload.metrics()) {
if (metric.name() == "Node Control/Rebirth" && metric.boolean_value()) {
publisher.rebirth();
}
}
});

std::expected< void, std::string > sparkplug::Subscriber::subscribe_all ( )
nodiscard

Subscribes to all Sparkplug B messages for the configured group.

Subscribes to the wildcard topic: spBv1.0/{group_id}/#

This receives all message types (NBIRTH, NDATA, NDEATH, DBIRTH, DDATA, DDEATH) from all edge nodes in the group.

Returns
void on success, error message on failure
Note
Must call connect() first.
The callback will be invoked for every message received.
